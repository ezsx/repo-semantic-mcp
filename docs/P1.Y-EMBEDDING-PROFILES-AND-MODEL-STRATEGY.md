# Спецификация: Embedding Profiles And Model Strategy

> **Версия:** 1.0
> **Дата:** 2026-03-08
> **Статус:** Approved
> **Основание:** результаты deep research по выбору embedding-моделей для `repo-semantic-mcp`

## 1) Проблема

Проекту нужны одновременно:
- стабильный CPU default для любой машины;
- рабочий GPU profile без поломки CPU пути;
- возможность сравнивать несколько embedding-моделей без смешивания несовместимых индексов;
- минимальные изменения MCP contract.

Дополнительный практический риск уже подтверждён на живом стенде:
- `BAAI/bge-m3` работает в TEI CUDA runtime;
- но cold-start ухудшается, потому что TEI грузит `pytorch_model.bin`, а не `safetensors`.

## 2) Утверждённые решения

1. `CPU default` остаётся на `intfloat/multilingual-e5-small`.
2. Для E5-family query/document formatting становится обязательной частью provider contract:
   - query: `query: {query}`
   - documents: `passage: {text}`
3. `GPU stable baseline` остаётся на `BAAI/bge-m3`, потому что этот профиль уже проверен end-to-end.
4. `GPU primary candidate` добавляется отдельно:
   - `Qwen/Qwen3-Embedding-0.6B`
5. `GPU fallback candidate` фиксируется в документации как следующий кандидат:
   - `Snowflake/snowflake-arctic-embed-l-v2.0`
6. Профили не имеют права делить одни и те же Qdrant collections, если отличаются:
   - embedding model;
   - query formatting;
   - document formatting;
   - prompt contract.
7. MCP API не меняется. Все различия живут только в:
   - env contract;
   - deploy profile;
   - embedding provider implementation.

## 3) Профили

### 3.1 CPU default

- backend: `tei_http`
- model: `intfloat/multilingual-e5-small`
- profile name: `cpu_e5`
- query template: `query: {query}`
- document prefix: `passage: `

### 3.2 GPU stable baseline

- backend: `tei_http`
- model: `BAAI/bge-m3`
- profile name: `gpu_bge_m3`
- query template: `{query}`
- document prefix: empty

### 3.3 GPU primary candidate

- backend: `tei_http`
- model: `Qwen/Qwen3-Embedding-0.6B`
- profile name: `gpu_qwen3`
- query template:
  - `Instruct: Retrieve relevant code and documentation snippets for repository search.\nQuery: {query}`
- document prefix: empty

## 4) Index isolation rule

Имя physical collection обязано включать:
- collection prefix;
- profile slug;
- model slug;
- scope;
- schema version.

Пример:
- `repo_semantic_cpu_e5_multilingual_e5_small_code_v1`
- `repo_semantic_gpu_bge_m3_bge_m3_docs_v1`
- `repo_semantic_gpu_qwen3_qwen3_embedding_0_6b_code_v1`

Это правило обязательно, потому что несовместимые query/document formatting иначе создают тихую corruption-класса проблему.

## 5) Deliverables

1. Provider contract поддерживает profile-aware formatting.
2. CPU env profile включает E5 prefixes.
3. GPU baseline profile остаётся рабочим.
4. Отдельный Qwen3 profile добавлен без замены baseline.
5. `index_status` показывает активный `index_profile`.

## 6) Acceptance criteria

1. CPU и GPU профили создают разные коллекции.
2. GPU baseline `bge-m3` продолжает успешно индексировать.
3. Qwen3 profile может быть поднят отдельным env без изменения MCP contract.
4. `index_status` явно показывает профиль и model.
5. Документация репозитория описывает:
   - CPU default;
   - GPU baseline;
   - GPU candidate strategy.
